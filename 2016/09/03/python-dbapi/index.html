    数据库连接对象Connection

    数据库驱动模块必须提供一个connect(parameters)函数来构造一个Connection对象。
    其中parameters是使用关键字参数，具体的参数是数据库相关的。

    Connection对象必须提供以下这些方法。

    .close() :

    Connection对象底层是用TCP实现的，所以不使用了肯定要关闭，免得占用系统资源。

    .commit() :

    如果数据库引擎支持事务，那么每次操作后，如果没有调用commit提交，那么数据是不会真正写入数据库的。

    .rollback() :

    如果数据库引擎支持事务，那么如何操作发送异常时，那么就可以使用rollback进行回回滚，记住commit之后，无法回滚了。比如像下面这样使用，这样就可以保证操作要么全做并且写入数据库，要么发生异常时进行回滚。

    .cursor(cursorclass) :

    获取Cursor对象，该对象是用来操作数据库的。
    我们可以通过制定cursorclass来回获取不同的cursor对象，这些cursor对象提供了相同的接口，但是表现了不同的行为。就MySQLdb来说，有四种不同的Cursor类，默认是使用第一种。

    1. Cursor : 默认就是生成该类的对象， 该对象将结果的每一行将存储在tuple中，并将结果结合存储在客户端。
    2. DictCursor : 该对象将结果的每一行存储在dict中，并将结果集合存储在客户端。
    3. SSCursor : 该对象将结果的每一行存储在tuple中，并将结果集合存储在服务器。
    4. SSDictCursor : 该对象将结果的每一行存储在dict中，并将结果集合存储在服务器。

    以上该类可以通过MySQLdb.cursors.*来获取。

    Cursor对象是用来执行sql语句，并获取执行结果的对象。
    Cursor对象必须提供以下这些方法和属性。

    .rowcount :

    这是属性是只读的， 用来表示.execute*()函数执行后产生(selecct)或者影响(update,insert)的结果的行数。 如果没有执行过.execute*()函数，那么rowcount的值是-1。

    .arraysize :

    .fetchmany(size=cursor.arraysize)，即.fetchmany()函数如果没有指定参数， 那么每次返回arraysize条记录。

    .callproc(procname[, parameters]) :

    这个方法是可选的，因为不是所有的数据库都支持存储过程。

    .close() :

    关闭cursor对象。

    .execute(operation[, parameters]) :

    执行sql语句，sql语句中可以使用占位符，然后在parameters中执行这些参数。如果有多个参数，参数放在必须放在列表中。使用何种占位符，不同的数据库驱动模块可能是不同的，但是数据库驱动模块必须提供一个全局变量paramstyle来指定使用何种占位符。

    MySQLdb模块的paramstyle = format，即使用C语言风格的占位符。

    .executemany(operation, seq_of_parameters) :

    执行多次sql语句，参数由sql_of_parameters指定。

    当执行多次查询语句时，可能返回一个或者多个结果集合，这是未定义的，不同的数据库模块可以自由实现。但是允许这样的行为： 执行operation时发现已经存在结果集合了，那么允许抛出一个异常。

    在MySQLdb模块中，如果使用将结果集存储在客户端的cursor对象执行多次查询语句时，即Cursor和DictCursor的对象时，只会返回最后一个查询的结果集合。 如果使用将结果存储在服务器的cursor对象执行多次查询语句时，即SSCURSOR和SSDictCursor对象时，会抛出Programingerror异常。
    不管是.execute()还是.executemany()，只要多次查询，就会发生上述所说的事情。解决方案就是通过数据库连接对象建立多个cursor进行查询那么没问题了。

    .fetchone() :

    从查询结果集中取出下一行，如果没有数据了，那么返回None。如果在调用该方法之前没有调用过.execute*()函数,那么会报错。

    .fetchmany(size=cursor.arraysize) :

    同.fetchone()，不过是一次返回多行。

    .fetchall() :

    同.fetchone()，不过是一次返回所有的查询结果。

    .nextset() :

    返回下一个结果集，为什么会产生多个结果集合呢， 见executemany(operation, seq_of_parameters。
    不是所有的数据库都支持多个结果集合的，所以这个方法是可选的。

    线程安全级别


    DBAPI规定数据库驱动模块必须含有全局变量threadsafety，用该变量指出该模块是属于哪一种线程安全的级别。

    1. 线程不同共享这个模块，即只能在一个线程中通过该模块访问数据库。
    2. 线程不能共享数据库连接，即每个线程要访问数据库时，只能通过建立新的连接访问。
    3. 线程能够共享一个数据库连接，但是不能共享cursor。
    4. 线程能够共享数据库连接和cursor。

    上面所说的共享，是你不用通过mutex就能够共享，也就是说其内部帮我们做到了线程安全。 那么我们可不可以通过mutex互斥访问数据库连接，使得线程之间能够共享threadsafety = 1的模块的数据库连接呢？答案是有可能可以，也有可能不可以，图片说有可能数据库连接依赖于某些外部资源，而这些外部资源不是我们能够控制的？？？
    MySQLdb的线程安全级别是threadsafety = 1。
