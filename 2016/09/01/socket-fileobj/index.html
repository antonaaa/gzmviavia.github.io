socket编程我觉得最基础最重要的就是做到正确的接收和发送数据，Python提供了两种方法接收和发送数据：socket对象和文件对象。

  • socket对象提供了操作系统的send()、sendto()、recv()、recvfrom()调用的接口。
  • 文件类对象提供了read()、readline()、readlines()、write()、writelines()这些典型好用的Python接口。

为了学到如何正确接收和发送数据，可以通过看socket模块的源码，从而得到如何实现read()、readline()等这些接口。

缓冲类型

创建一个文件对象时，需要通过参数bufsize指定其缓冲类型，缓冲分为全缓冲、行缓冲和不带缓冲。

  • bufsize < 0 : 表示使用使用系统默认大小的缓冲区。
  • bufsize == 0 : 表示不使用缓冲
  • bufsize == 1 : 表示使用行缓冲
  • bufsize > 1 : 表示使用bufsize大小的缓冲区

缓冲对于读操作来说，表示一次从socket中读取多大的数据。
缓冲对于写操作来说，只有要写的数据超过了缓冲的限制才会真正地通过socket发送数据。

读操作

read(self, size = -1)

read()函数的作用是从socket中读取size个字节的数据，size=-1表示读取无限的数据，recv()函数返回’’，recv()函数返回’’，表示连接发生了异常，或者对方关闭了写。

当size=-1时，不管指定的是何种缓冲，将使用最大的缓冲大小从recv()函数中读取数据，因为使用小缓冲是毫无意义的，反而会浪费时间。
当size>0时，将会使用不够的字节数从recv()函数中读取数据。
